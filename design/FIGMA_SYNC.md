# Почему бывают несостыковки с Figma и как их убрать

## Причины расхождений

### 1. **Разные источники правды**
- В коде используются **приблизительные** значения (например, `max-width: 320px` для инпутов из общих стилей), а в Figma — точные размеры карточки (448px) и полная ширина полей/кнопки внутри неё.
- **Решение:** для экранов из Figma завести отдельные классы (например, `.auth-form`, `.login-card`) и задавать ширины явно: карточка `width: 100%`, форма и кнопка — `width: 100%` без `max-width`, чтобы они тянулись на всю ширину карточки.

### 2. **Глобальные стили перебивают макет**
- Глобальные правила (`input { max-width: 320px }`) применяются везде и сужают поля на странице логина/регистрации.
- **Решение:** в контексте карточки авторизации переопределять: `.auth-form input { max-width: none; width: 100%; }`, то же для кнопки и обёртки пароля.

### 3. **Размер логотипа и типографика**
- В Figma лого и заголовки страниц: **24px, weight 700** (см. figma-design-tokens.md). В коде могло быть 30px.
- **Решение:** использовать токены из макета: `font-size: 24px` для лого, в CSS можно завести переменную `--logo-font-size: 24px` и использовать её в `.login-logo`, `.register-logo`, заголовках.

### 4. **Логотип = иконка + текст**
- В Figma в шапке и на карточках логина/регистрации часто идёт **значок (два переплетённых кольца)** и текст «Pairly», а не только текст.
- **Решение:** вынести логотип в компонент (например, `PairlyLogo.tsx`): иконка + текст «Pairly»; в шапке — белый цвет, на карточках — градиент на текст и `currentColor` на иконку.

### 5. **Нет единого источника размеров**
- Отступы, радиусы, высоты кнопок в коде могут быть «на глаз», а в Figma заданы в пикселях.
- **Решение:** выписать из Figma все размеры для ключевых экранов (логин, регистрация, профиль, списки) в один файл (например, `figma-design-tokens.md`) и в коде использовать только их: padding карточки, высота кнопки, радиус 8px/12px и т.д.

### 6. **Шрифт и межбуквенное расстояние**
- В макете указан Inter; если в проекте другой шрифт или другое начертание, визуал будет отличаться.
- **Решение:** в `layout.tsx` подключать Inter (например, через `next/font/google`) и применять его к `body`; при необходимости задать `letter-spacing` из Figma для лого/заголовков.

## Что сделано в коде для уменьшения расхождений

1. **Логин/регистрация**
   - Карточка: `max-width: var(--app-max)` (448px), `width: 100%`.
   - Форма: `width: 100%`; поля и кнопка: `width: 100%`, `max-width: none` в контексте `.auth-form`, чтобы кнопка «Войти» и поля были на всю ширину карточки.
   - Лого: 24px по Figma, добавлен компонент с иконкой (два кольца) + текст «Pairly» на карточках и в шапке.

2. **Единый источник токенов**
   - Цвета, градиент, `--app-max`, отступы заданы в `globals.css` и в `design/figma-design-tokens.md`; новые экраны сверять с этим файлом.

3. **Компонент логотипа**
   - `PairlyLogoMark` (иконка) и `PairlyLogoWithText` (иконка + «Pairly») используются в `AppLayout`, на страницах логина и регистрации.

## Как дальше держать дизайн в синхронизации с Figma

1. **Перед версткой нового экрана:** открыть макет в Figma, выписать в `figma-design-tokens.md` размеры (отступы, размеры шрифтов, высоты кнопок, радиусы), затем в коде использовать только эти значения.
2. **Проверка по слоям:** в Figma включить отображение линеек и размеров, сравнить с DevTools (computed styles) по ключевым блокам (карточка, форма, кнопка).
3. **Экспорт ресурсов:** иконки и логотип экспортировать из Figma (SVG) и подключать в коде, чтобы не рисовать «похоже».
4. **Один файл — один экран:** в `DESIGN_COMPARISON.md` помечать, какие экраны уже сверены с макетом (по элементам из списка), и при изменении макета обновлять и код, и чеклист.
